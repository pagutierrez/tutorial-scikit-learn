Ejemplos de uso con el dataset iris¶

Vamos a utilizar un ejemplo tpico en machine learning que es la base de datos iris. En esta base de datos hay tres clases a predecir, que son tres especies distintas de la flor iris, de manera que, para cada flor, se extraen cuatro medidas o variables de entrada (longitud y ancho de los ptalos y los spalos, en cm). Las tres especies a distinguir son iris setosa, iris virginica e iris versicolor.

Lectura de datos¶

Como ya hemos comentado, para la lectura de datos haremos uso de Pandas. Esta librera tiene un mtodo read_csv que nos va a permitir leer los datos desde un fichero de texto csv.

In [1]:
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

from sklearn import neighbors
from sklearn import preprocessing

Con estas lneas, importamos la funcionalidad necesaria para el ejemplo. pandas nos permitir leer los datos, numpy nos va a permitir trabajar con ellos de forma matricial, matplotlib nos permite hacer representaciones grficas y, de la librera scikit-learn, en este caso, utilizaremos un mtodo de clasificacin basado en los vecinos ms cercanos y algunas funciones de preprocesamiento.

El mtodo read_csv de pandas permite dos modos de trabajo: que el propio fichero csv tenga una fila con los nombres de las variables o que nosotros especifiquemos los nombres de las variables en la llamada. En este caso, vamos a utilizar la segunda aproximacin. De esta forma, creamos un array con los nombres de las variables:

In [2]:
nombre_variables = ['longitud_sepalo', 'ancho_sepalo', 'longitud_petalo', 'ancho_petalo', 'clase']

y leemos el dataset con:

In [3]:
iris = pd.read_csv('data/iris.csv', names = nombre_variables)

iris es un objeto de la clase DataFrame de pandas. Tambin podramos haber obviado el nombre de las columnas estableciendo header=None, de forma que read_csv le hubiera asignado un nombre por defecto.

La sintaxis de indexacin en un ndarray es la siguiente:

  • array[i,j]: accede al valor de la fila i columna j.
  • array[i:j,k]: devuelve otro ndarray con la submatriz correspondiente a las filas desde la i hasta la j-1 y a la columna k.
  • array[i:j,k:l]: devuelve otro ndarray con la submatriz correspondiente a las filas desde la i hasta la j-1 y a las columnas desde la k hasta la l.
  • array[i:j,:]: devuelve otro ndarray con la submatriz correspondiente a las filas desde la i hasta la j-1 y todas las columnas.
  • array[:,i:j]: devuelve otro ndarray con la submatriz correspondiente a todas las filas y a las columnas desde la k hasta la l. De esta forma:

Vectorización de operaciones¶

En scikit-learn, al igual que en otros lenguajes de programacin como R o Matlab, debemos intentar, siempre que sea posible, vectorizar las operaciones. Esto es utilizar operaciones matriciales en lugar de bucles que recorran los arrays. La razn es que este tipo de operaciones estn muchos ms optimizadas y que el proceso de referenciacin de arrays puede consumir mucho tiempo.

División de datos en entrenamiento y test¶

Aunque a veces nos proporcionan los datos ya divididos en los conjuntos de entrenamiento y test, conviene saber como podramos realizar esta divisin. El siguiente cdigo muestra una funcin que divide los datos de forma aleatoria, utilizando operaciones vectorizadas:

In [41]:
def dividir_ent_test(dataframe, porcentaje=0.6):
    """ 
    Funcin que divide un dataframe aleatoriamente en entrenamiento y en test.
    Recibe los siguientes argumentos:
    - dataframe: DataFrame que vamos a utilizar para extraer los datos
    - porcentaje: porcentaje de patrones en entrenamiento
    Devuelve:
    - train: DataFrame con los datos de entrenamiento
    - test: DataFrame con los datos de test
    """
    mascara = np.random.rand(len(dataframe)) < porcentaje
    train = dataframe[mascara]
    test = dataframe[~mascara]
    return train, test

Labores de preprocesamiento¶

Sin embargo, scikit-learn no acepta cadenas como parmetros de las funciones, todo deben de ser nmeros. Para ello, nos podemos valer del objeto sklearn.preprocessing.LabelEncoder, que nos transforma automticamente las cadenas a nmeros. La forma en que se utiliza es la siguiente:

In [44]:
label_e = preprocessing.LabelEncoder()
label_e.fit(train_outputs_iris)
train_outputs_iris_encoded = label_e.transform(train_outputs_iris)
test_outputs_iris_encoded = label_e.transform(test_outputs_iris)

Como podis observar, primero se crea el LabelEncoder y luego se "entrena" mediante el mtodo fit. Para un LabelEncoder, "entrenar" el modelo es decidir el mapeo que vimos anteriormente, en este caso:

  • Iris-setosa -> 0
  • Iris-versicolor -> 1
  • Iris-virginica -> 2

Una vez entrenado, utilizando el mtodo transform del LabelEncoder, podremos transformar cualquier ndarray que queramos (hubiramos tenido un error si alguna de las etiquetas de test no estuviera en train). Esta estructura (mtodo fit ms mtodo transform o predict) se repite en muchos de los objetos de scikit-learn.

Hay muchas ms tareas de preprocesamiento que se pueden hacer en scikit-learn. Consulta el paquete sklearn.preprocessing.
